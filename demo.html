Smarter alarms

Lunifer is an adaptive iOS alarm app that uses your WHOOP sleep need to determine the ideal wake time — then adjusts it automatically if your bedtime shifts.

Who uses Lunifer

Health-driven sleepers who already wear WHOOP

Lunifer's users are students and working professionals who take sleep seriously. They already track recovery with WHOOP — Lunifer closes the loop by turning that recovery data into an alarm time that actually responds to it.

Students Working professionals Athletes Remote workers WHOOP subscribers
The core problem

Fixed alarms ignore what your body actually needs

Most alarm apps fire at the same time regardless of sleep quality, strain, or debt. A user who went to bed an hour late still gets woken at 7:00 AM as if nothing happened. Lunifer and WHOOP solve this together — WHOOP measures what the body needs, Lunifer acts on it.

User journey

How WHOOP flows through Lunifer

WHOOP data touches Lunifer at three distinct moments — onboarding, nightly alarm calculation, and real-time adaptive rescheduling.

1

Onboarding — connect your WHOOP

During the sleep duration step of Lunifer's setup survey, users see a "Let my WHOOP decide" option alongside manual time entry. Tapping it launches WHOOP's OAuth 2.0 + PKCE authorization flow via ASWebAuthenticationSession. Once authorized, Lunifer fetches the user's personalized sleep need from the WHOOP API and stores it as their target sleep duration. The survey advances automatically.

2

Nightly alarm calculation

Each evening, Lunifer calculates tomorrow's alarm time using the formula: alarm = bedtime + WHOOP sleep need. If WHOOP says the user needs 8h 45m tonight (elevated strain, partial sleep debt), the alarm is set accordingly — not at a hardcoded 7:00 AM. The user sees "recommended to you via WHOOP" on their Sleep Insights card, making the source transparent.

3

Adaptive rescheduling overnight

If the user falls asleep later than their target bedtime, Lunifer's adaptive engine detects the sleep onset and pushes the alarm forward by the same delta — preserving the full WHOOP-recommended sleep duration. If they fall asleep early, the alarm pulls earlier. The WHOOP sleep need is the anchor for all of these adjustments.

4

Morning — silent resync

When the user opens Lunifer the next day, WhoopManager.refreshIfNeeded() silently fetches a fresh sleep need if the last sync was more than 12 hours ago. The Sleep Insights card updates automatically. No user action required.

5

Settings — manage the connection

Users can connect or disconnect WHOOP at any time from the Sleep & Wearables screen in Settings. Disconnecting gracefully reverts the sleep recommendation to the user's manual setting or Lunifer's age-based estimate. The connection state is always visible.

API Usage

Data Lunifer reads from WHOOP

Lunifer requests the minimum scopes necessary — read:sleep and read:cycles. We read one field: the user's personalized sleep need. Nothing else is stored or transmitted beyond what's required to set tomorrow's alarm.

🔋

Baseline sleep need

The minimum sleep the user needs on a typical night, based on their physiology.

baseline_milli
💤

Sleep debt adjustment

Additional sleep recommended to recover from cumulative under-sleeping.

need_from_sleep_debt_milli
⚡

Strain adjustment

Extra sleep needed after high physical or cardiovascular strain days.

need_from_recent_strain_milli
🌙

Nap credit

Sleep need reduction when the user took a nap, so the alarm isn't pushed unnecessarily late.

need_from_recent_nap_milli
Sleep need formula
total_ms = baseline_milli + need_from_sleep_debt_milli + need_from_recent_strain_milli − need_from_recent_nap_milli
sleep_need_hours = clamp(total_ms / 3,600,000, min=5, max=12)
Endpoint: GET /developer/v1/cycle?limit=1  ·  Called once per day, or when the app returns to foreground after 12+ hours
Privacy & Data Handling

How Lunifer handles WHOOP data

Lunifer treats WHOOP data with the same care WHOOP users expect. We store the minimum, transmit securely, and give users full control.

🔐

Tokens never touch the app

WHOOP OAuth tokens are exchanged and stored exclusively in a Cloudflare Worker backend (KV storage, keyed by Firebase UID). The iOS app never holds a raw WHOOP access or refresh token.

📦

Minimal data stored on-device

Lunifer stores only three values locally: whoopConnected (Bool), whoopRecommendedSleepHours (Double), and whoopLastSyncDate (Date). No raw WHOOP API responses are persisted.

🗑️

Full deletion on disconnect or account delete

Disconnecting WHOOP calls the backend disconnect endpoint (which removes tokens from KV) and clears all local WHOOP values. Deleting a Lunifer account also purges the backend token entry.

👤

User-controlled at all times

Users can disconnect WHOOP from the Sleep & Wearables settings screen at any time. The connection state and data source are always visible on the Sleep Insights card and in Settings.

📄

Disclosed in privacy policy

Lunifer's public privacy policy documents WHOOP as an optional third-party integration, what data is requested, how it is used, and how users can revoke access. The policy is hosted publicly and linked from the WHOOP developer dashboard.
